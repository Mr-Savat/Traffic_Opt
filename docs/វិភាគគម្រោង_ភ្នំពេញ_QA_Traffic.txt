────────────────────────────────────────────────────
  PAPER 1 — Neukart et al. (2017) — Volkswagen + D-Wave
  "Traffic flow optimization using a quantum annealer"
  arXiv: 1708.01625 | Frontiers in ICT
────────────────────────────────────────────────────
  ▶ PIONEER paper — ដំបូងគេ! ប្រើ Real GPS data Beijing
  ▶ Problem: Route assignment (Car routing) មិនមែន Signal timing
  ▶ Hardware: D-Wave 2X QPU (Hybrid classical + quantum)
  ▶ Data: Real Beijing GPS trajectory data
  ▶ Result: QA viable for real-time car routing
  ▶ Limitation ចំពោះ Project អ្នក:
    - Cars only (NO motorcycle)
    - Route optimization (NOT signal control)
    - Grid city (NOT irregular Phnom Penh topology)

────────────────────────────────────────────────────
  PAPER 2 — Hussain et al. (2019) — Quantum Info. Processing
  "Optimal Control of Traffic Signals using Quantum Annealing"
  arXiv: 1912.07134 | DOI: 10.1007/s11128-020-02815-1
────────────────────────────────────────────────────
  ▶ QUBO formulation + D-Wave QPU (Hybrid approach)
  ▶ Problem: Signal timing on artificial grid network
  ▶ Data: Synthetic/simulated (no real sensor)
  ▶ Workflow: Periodic multi-instance solving (time-dependent)
  ▶ Result: QA competitive vs classical for signal control
  ▶ Limitation ចំពោះ Project:
    - Cars only (NO motorcycle)
    - Synthetic grid (NOT real-world topology)
    - No heterogeneous vehicle types

────────────────────────────────────────────────────
  PAPER 3 — Inoue et al. (2021) — Nature Scientific Reports
  "Traffic Signal Optimization on a Square Lattice with QA"
  arXiv: 2003.07527 | DOI: 10.1038/s41598-021-82740-0
────────────────────────────────────────────────────
  ▶ Large-scale 50×50 Grid city (2500 intersections)
  ▶ Ising Hamiltonian: H(σ) = σᵀJσ + hᵀσ
  ▶ Hardware: D-Wave 2000Q (64 qubits/subproblem)
  ▶ Partitioning: Metis software (42 groups)
  ▶ Baseline: QA > SA > Local Control (when α > 0.8)
  ▶ Limitation ចំពោះ Project:
    - Cars only (NO motorcycle)
    - Perfect grid (NOT Phnom Penh irregular roads)
    - Synthetic data (no real-world validation)

────────────────────────────────────────────────────
  PAPER 4 — Singh et al. (2022) — IEEE SNPD Conference
  "QA Approach for Optimal Real-time Traffic Control using QUBO"
  arXiv: 2403.09023 | DOI: 10.1109/SNPD51163.2021.9704985
  ⭐ MOST RELEVANT TO YOUR PROJECT
────────────────────────────────────────────────────
  ▶ REAL intersection (Dongda-Keyuan, Taiwan)
  ▶ HETEROGENEOUS: Cars + Scooters (2 vehicle classes)
  ▶ C-QUBO (constrained) + QUBO (unconstrained) approach
  ▶ Dynamic signal duration (variable green/red time)
  ▶ Data: Real traffic sensor data (Taiwan)
  ▶ Result: QUBO > C-QUBO > Fixed Cycle method
  ▶ Limitation ចំពោះ Project:
    - Scooter = 20% traffic (ក្នុង Taiwan)
    - Phone Penh: Motorcycle = 58-80% dominant!
    - Regular grid intersection (NOT Phnom Penh irregular)
    - 2 classes only (Car + Scooter)
    - Developed country infrastructure (Taiwan)

────────────────────────────────────────────────────
  PAPER 5 — Shikanai, Ohzeki, Tanaka (2023/2025) — NEW!
  "QUBO Formulation for Traffic Signal Optimization on
   Real-World Maps" (T-junctions + Multi-forked roads)
  arXiv: 2308.14462 | Journal of Physical Society Japan, 2025
────────────────────────────────────────────────────
  ▶ KEY NOVELTY: QUBO extended ទៅ T-junctions + multi-forked!
    (ប្រែ: ការ crossroads ច្រើន-ជ្រុង — ស្ទើរ fit ភ្នំពេញ!)
  ▶ Validation: SUMO simulator + Real Aomori City (Japan) GIS data
  ▶ Comparison: D-Wave vs Gurobi (classical solver)
  ▶ Result: QUBO reduces red light waiting ~28.9% (high traffic)
  ▶ Important Finding: D-Wave SLOWER than Gurobi Optimizer!
    → QA hardware limitations still exist in current generation
  ▶ Relevance ចំពោះ Project:
    + T-junction/multi-fork = VERY similar to Phnom Penh road shape
    + SUMO simulator approach = អ្នកអាចប្រើ SUMO ដូចគ្នា!
    + Honest about QA limitations = Good academic model
  ▶ Limitation:
    - Cars only (NO motorcycle)
    - Japan (Developed country — NOT SE Asia context)

────────────────────────────────────────────────────
  PAPER 6 — Inoue, Yamashita et al. (2024) — NEW! NEWEST!
  "AMPIC: Adaptive Model Predictive Ising Controller"
  arXiv: 2406.03690 | Submitted June 2024 (Latest!)
  (Toyota Central R&D Labs — GitHub available!)
────────────────────────────────────────────────────
  ▶ AMPIC = Adaptive Model Predictive Ising Controller
  ▶ Combines: Model Predictive Control (MPC) + Ising/QA
  ▶ LARGE-SCALE: Realistic city road network (not just grid)
  ▶ Tool: SUMO microscopic traffic simulator
  ▶ Ising Solver: D-Wave QA = near-optimal at low cost
  ▶ Result: Faster cruising speed + Less waiting + Less CO2
  ▶ CODE: Available on GitHub (Toyota CRDL)
  ▶ Relevance ចំពោះ Project:
    + MPC + Ising = New approach អ្នកអាចពិចារណា extend!
    + SUMO validation = framework ដ្ដ ‌ reusable!
    + Large-scale urban = closer to real Phnom Penh network
  ▶ Limitation:
    - Cars only (NO motorcycle)
    - Generic city model (NOT Phnom Penh)

ផ្នែក ៨: Suggested Methodology (ផ្អែកលើ synthesis ៦ paper)
════════════════════════════════════════════════════════════════════

  STEP 1: Define Problem (Week 1-2)
  ─────────────────────────────────
  • ជ្រើស intersection: ភ្នំពេញ (e.g., Monivong x Norodom)
    ឬ ប្រើ synthetic T-junction + multi-fork (like Shikanai)
  • Define vehicle classes:
    Class A = Motorcycle (dominant, weight = 0.5 PCU)
    Class B = Car/SUV (weight = 1.0 PCU)
    Class C = Tuk-tuk/Minibus (weight = 1.5 PCU)
  • Define signal phases: NS_green=1, EW_green=0 (like Hussain)

  STEP 2: QUBO Formulation (Week 3-5)
  ────────────────────────────────────
  • Build on Singh (2022) QUBO → extend to 3 vehicle classes:

    Q_motorcycle(x) = Σ w_m × queue_m(x)   [dominant weight]
    Q_car(x)        = Σ w_c × queue_c(x)   [secondary]
    Q_tuk(x)        = Σ w_t × queue_t(x)   [tertiary]
    H(x) = Q_motorcycle + Q_car + Q_tuk + P × constraints

  • Constraint: conflict-free signal (no simultaneous green)
  • Dynamic duration: variable green time (like Singh C-QUBO/QUBO)
  • For T-junctions: use Shikanai formulation (extend it)

  STEP 3: SUMO Simulation (Week 6-8)
  ───────────────────────────────────
  • SUMO + Sublane Model (SL2015) for motorcycle weaving
  • Generate synthetic traffic:
    70% motorcycle + 20% car + 10% tuk-tuk
    (justify with Phnom Penh modal share data)
  • Traffic scenarios:
    Scenario A: Rush hour (AM peak 7-9am)
    Scenario B: Rush hour (PM peak 5-7pm)
    Scenario C: Off-peak balanced
    Scenario D: Heavy moto "wave" at signal start

  STEP 4: Solve with QA/SA (Week 9-10)
  ──────────────────────────────────────
  • Primary Solver: neal.SimulatedAnnealingSampler() (free, local)
  • Secondary: D-Wave LeapHybridSampler (academic access)
  • Baseline 1: Fixed Cycle method (like Singh)
  • Baseline 2: Webster's method (classical traffic engineering)
  • Compare: QA vs SA vs Fixed vs Webster

  STEP 5: Evaluate + Write (Week 11-12)
  ───────────────────────────────────────
  • Metrics:
    - Average Queue Length (vehicles/phase)
    - Average Waiting Time (seconds/vehicle)
    - Motorcycle-specific throughput (vehicles/min)
    - Signal switch frequency (stability check)
  • Report structure (follow Singh paper structure):
    Abstract → Intro → Lit Review → Methodology →
    Results → Discussion → Limitation → Conclusion
