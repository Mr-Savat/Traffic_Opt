# Project Summary Note (ឯកសារសង្ខេបគម្រោងច្បាស់លាស់) 🎓

ឯកសារនេះធ្វើការចងក្រងនូវរាល់កិច្ចការទាំងឡាយដែលពួកយើងបានសម្រេចរួមគ្នាតាំងពីដើមរហូតដល់ចប់ សម្រាប់គម្រោង **"Quantum-Enhanced Traffic Signal Optimization for Heterogeneous Motorcycle-Dominant Traffic in Phnom Penh"**។ អ្នកអាចប្រើឯកសារនេះទុកជា Checklist ឬជំនួយស្មារតីពេលធ្វើ Presentation បាន។

---

## 1. Project Objective (គោលបំណងចម្បង)
ដើម្បីបង្កើតប្រព័ន្ធស្តុបចរាចរណ៍វៃឆ្លាត ផ្តោតការសង្កត់ធ្ងន់ទៅលើ **ចរាចរណ៍ដែលពោរពេញដោយម៉ូតូ (Motorcycle-Dominant)** ដូចនៅទីក្រុងភ្នំពេញ។ 
យើងបានធ្វើការប្រៀបធៀបរវាងការប្រើប្រាស់ "ស្តុបវដ្តថេរធម្មតា (Fixed Cycle)" ជាមួយនឹង "ស្តុបឆ្លាតវៃ QUBO" ដែលដោះស្រាយដោយ Simulated Annealing (Quantum Method)។

## 2. Tools & Libraries Installed (បច្ចេកវិទ្យាដែលបានបំពាក់)
- **SUMO Simulator (និង TraCI):** សម្រាប់បង្កើតចរាចរណ៍ Virtual (មានផ្លូវ និងរថយន្តដើរជាក់ស្តែង)។
- **Python:** សម្រាប់សរសេរកូដបញ្ជាកណ្តាល។
- **D-Wave Ocean SDK (`dimod`, `neal`):** សម្រាប់បម្លែងបញ្ហាស្ទះចរាចរណ៍ទៅជាក្បួនគណិតវិទ្យា Quantum លំដាប់ខ្ពស់ (QUBO) ។
- **Pandas & Matplotlib:** សម្រាប់វិភាគ និងគូរក្រាហ្វ។

## 3. Directory & File Structure (តួនាទីឯកសារនិមួយៗដែលយើងបានសរសេរ)

### 🔹 Phase 1: SUMO Network (ការបង្កើតប្លង់ចរាចរណ៍)
- `phnom_penh.nod.xml` & `phnom_penh.edg.xml`: ជាកូដបង្កើតចំណុចប្រសព្វផ្លូវខ្វែង ដែលមាន ៣ គន្លងសងខាង។
- `vehicles.rou.xml`: ជាអ្នកកំណត់ប្រភេទយានយន្ត។ ពិសេសបានបំពាក់មុខងារ `SL2015` ឲ្យម៉ូតូអាចវ៉ៀបជ្រៀតគ្នាបាន (Sublane sharing behaviour ដូចជិះនៅភ្នំពេញមែន)។
- `flows.rou.xml`: ជាអ្នកកំណត់ចំនួនចរាចរណ៍ (Rush Hour SC-1: ម៉ូតូ 700, ឡាន 150, តុកតុក 60 គ្រឿងក្នុង ១ ម៉ោង/ទិស)។
- `build_network.bat`: ជាឯកសារវេទមន្តសម្រាប់ភ្ជាប់កូដខាងលើទាំងអស់ បញ្ចូលគ្នាបង្កើតចេញជាប្លង់ផ្លូវផ្លូវការ `phnom_penh.net.xml` សម្រាប់ឲ្យ SUMO ស្គាល់។
- `phnom_penh.sumocfg`: ឯកសារ Setting មេសម្រាប់បញ្ជា SUMO ។

### 🔹 Phase 2: Quantum QUBO Model (ប្រព័ន្ធខួរក្បាល Artificial Quantum AI)
បានបង្កើតនៅក្នុង Folder ឈ្មោះ `NU_Project_Traffic_Opt`:
- `qubo_builder.py`: ជាអ្នកដើរតួគិតទម្ងន់ចរាចរណ៍ពិតប្រាកដនៅភ្នំពេញ (ម៉ូតូ=0.60, ឡាន=0.30, តុកតុក=0.10) ។ វាគណនាថាបើមានម៉ូតូច្រើនកកស្ទះ វាជួយឲ្យពិន្ទុទិសនោះធ្ងន់ធ្ងរជាងគេ។
- `solvers.py`: អ្នកទទួលយកបញ្ហាពី QUBO មកដោះស្រាយដោយប្រើ `SimulatedAnnealingSampler` របស់ D-Wave ។ វាជាអ្នកប្រាប់ថា គួរតែ "បើកភ្លើងខៀវទិសជើង-ត្បូង (1)" ឬ "កើត-លិច (0)" ដើម្បីឲ្យការកកស្ទះសរុបមានចំនួនតិចតួចបំផុត!

### 🔹 Phase 3: The Engine Bridge (ស្ពានតភ្ជាប់)
- `main_simulation.py`: កូដដ៏សំខាន់បំផុតដែលតភ្ជាប់ Python និង SUMO ។:
  * រាល់ ៣០ វិនាទីម្តង វានឹងស្កេនមើលរថយន្តដែលកំពុងឈប់ស្តុប។
  * វាបញ្ជូនទិន្នន័យស្ទះនោះទៅកូដ QUBO។ 
  * ទទួលបានលទ្ធផលវិញ វានឹងបញ្ជា SUMO ឲ្យប្តូរភ្លើងស្តុបភ្លាមៗ។
  * ឯកសារនេះក៏ជាអ្នកកត់ត្រាលទ្ធផលផងដែរ (បង្កើតបានជាឯកសារ `results_Fixed_SC1.csv` និង `results_SA_SC1.csv`)។

### 🔹 Phase 4: Data Analysis (ជំពូកវិភាគលទ្ធផល)
- `analyze.py`: ប្រើសម្រាប់អានទិន្នន័យ CSV ខាងលើ ហើយផលិតចេញជារូបភាពប្រៀបធៀប។ 
- រូបភាពលទ្ធផលចំនួន ២: `average_queue_comparison.png` និង `queue_over_time.png` បង្ហាញឲ្យឃើញថាប្រព័ន្ធរបស់យើង (QUBO) **ជួយសម្រួលការកកស្ទះបានដល់ទៅ ៣០.៩១%** ល្អជាងស្តុបធម្មតាដាច់!
