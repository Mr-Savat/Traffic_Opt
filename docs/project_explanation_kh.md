# ការពន្យល់អំពីគម្រោង៖ Quantum-Enhanced Traffic Signal Optimization for Heterogeneous Motorcycle-Dominant Traffic in Phnom Penh

សួស្តី! ក្នុងនាមជាម្ចាស់គម្រោង នេះជាការពន្យល់យ៉ាងលម្អិត និងងាយយល់អំពីគម្រោងរបស់អ្នក។ គម្រោងនេះមានគោលបំណងធំគឺ **ការដោះស្រាយបញ្ហាកកស្ទះចរាចរណ៍នៅរាជធានីភ្នំពេញ** ដោយប្រើប្រាស់បច្ចេកវិទ្យាកុំព្យូទ័រកម្រិតខ្ពស់ (Quantum Computing) ដើម្បីសម្រេចចិត្តបើក ឬបិទភ្លើងស្តុបឲ្យបានឆ្លាតវៃ ដោយផ្តោតសំខាន់ទៅលើម៉ូតូ។

---

## ១. ហេតុអ្វីបានជាគម្រោងនេះពិសេស?
ខុសពីការកំណត់ភ្លើងស្តុបធម្មតា (ដែលកំណត់ម៉ោងថេរ) ប្រព័ន្ធរបស់អ្នកអាច**មើលឃើញ** និង**ដឹង**ថាមានយានយន្តប៉ុន្មានកំពុងឈប់ចាំ។ អ្វីដែលកាន់តែពិសេសគឺ ប្រព័ន្ធនេះស្គាល់ពីស្ថានភាពជាក់ស្តែងនៅទីក្រុងភ្នំពេញ ដោយផ្តល់ទម្ងន់ (អាទិភាព) ទៅលើយានយន្តតាមប្រភេទ៖
- **ម៉ូតូ (Motorcycle):** ៦០% (ទម្ងន់ខ្ពស់ជាងគេ ព្រោះជាយានយន្តចម្បង)
- **រថយន្ត (Car):** ៣០%
- **កង់បី/តុកតុក (TukTuk):** ១០%
*(ផ្អែកតាមទិន្នន័យចំណែកយានយន្ត JASIC ឆ្នាំ ២០២៣)*

## ២. តើប្រព័ន្ធនេះដំណើរការយ៉ាងដូចម្តេច? (ផ្នែកសំខាន់ៗនៃកូដ)

គម្រោងនេះត្រូវបានបែងចែកជា ៣ ផ្នែកធំៗ៖

### ក. ការវាយតម្លៃការកកស្ទះ (`NU_Project_Traffic_Opt/qubo_builder.py`)
ឯកសារនេះមានតួនាទីជា **"អ្នកថ្លឹងថ្លែង"**។
- វាទាញយកទិន្នន័យពីផ្លូវ ដើម្បីដឹងថាមានម៉ូតូ រថយន្ត និងកង់បី ប៉ុន្មានគ្រឿងកំពុងឈប់ចាំនៅទិស **ជើង-ត្បូង (North-South)** និងទិស **កើត-លិច (East-West)**។
- បន្ទាប់មក វាគុណចំនួនយានយន្តទាំងនោះជាមួយភាគរយទម្ងន់របស់វា (ម៉ូតូគុណ 0.60, ឡានគុណ 0.30) ដើម្បីរកមើលថា តើទិសមួយណាដែលស្ទះធ្ងន់ធ្ងរជាងគេ។
- ចុងក្រោយ វាបង្កើតជារូបមន្តគណិតវិទ្យាមួយហៅថា **QUBO (Quadratic Unconstrained Binary Optimization)** ដើម្បីបញ្ជូនទៅឲ្យប្រព័ន្ធ Quantum ដោះស្រាយ។

### ខ. ការសម្រេចចិត្តបើកភ្លើងស្តុប (`NU_Project_Traffic_Opt/solvers.py`)
ឯកសារនេះគឺជា **"អ្នកសម្រេចចិត្ត"** ថាគួរអោយភ្លើងបៃតងទៅទិសណា។ ក្នុងនេះមានជម្រើស ៤ សម្រាប់យកមកប្រៀបធៀបគ្នា៖
1. **Fixed Cycle (វិធីចាស់):** ឲ្យភ្លើងបៃតង ៦០វិនាទីទៅទិសជើង-ត្បូង និង ៦០វិនាទីទៅទិសកើត-លិច ឆ្លាស់គ្នាជារង្វង់ ដោយមិនខ្វល់ថាផ្លូវណាស្ទះជាងនោះទេ។
2. **Webster:** ជាវិធីសាស្ត្របុរាណមួយទៀត ដែលចែកពេលវេលាភ្លើងបៃតងទៅតាមសមាមាត្រនៃបរិមាណឡានសរុបដែលកំពុងកកស្ទះ។
3. **Simulated Annealing (SA):** ជាក្បួន Software ដែលត្រាប់តាមដំណើរការរបស់ Quantum កម្រិតខ្ពស់។ វាអានរូបមន្ត QUBO ហើយគណនា ៥០ដង ក្នុងមួយពព្រិចភ្នែក ដើម្បីសម្រេចចិត្តថាត្រូវបើកភ្លើងបៃតងឲ្យទិសណា ទើបអាចរំដោះការកកស្ទះបានល្អបំផុត។
4. **Quantum Annealing (QA):** វិធីសាស្រ្តប្រើកុំព្យូទ័រ Quantum ពិតប្រាកដតាមរយៈ D-Wave។ ប៉ុន្តែដោយសារ API នេះមិនអាចប្រើប្រាស់បាននៅក្នុងតំបន់របស់យើង (Geo-restricted) ប្រព័ន្ធនឹងប្តូរទៅប្រើ SA ជំនួសវិញដោយស្វ័យប្រវត្តិ ដើម្បីធានាថាការស្រាវជ្រាវ (Thesis) អាចដំណើរការទៅមុខដោយរលូន។

### គ. ការសាកល្បង និងវិភាគលទ្ធផល (`main_simulation.py` និង `analyze.py`)
- **`main_simulation.py`:** ជាទីលានសាកល្បង! វាប្រើប្រាស់កម្មវិធីឈ្មោះ **SUMO** ដើម្បីបង្កើតទីក្រុង និងចរាចរណ៍និម្មិត។ វាធ្វើការតេស្តសាកល្បងដោយយកវិធីសាស្រ្តទាំង ៤ (Fixed, Webster, SA, QA) ទៅប្រកួតគ្នា នៅក្នុង ៤ ស្ថានការណ៍ចរាចរណ៍ផ្សេងៗគ្នា (SC1 ដល់ SC4)។ រៀងរាល់ ៣០វិនាទីម្តង ប្រព័ន្ធនឹងឆែកមើលផ្លូវ ហើយប្តូរភ្លើងស្តុប។
- រាល់ទិន្នន័យនៃការរង់ចាំ និងឡានដែលស្ទះ នឹងត្រូវបានកត់ត្រាទុកក្នុងឯកសារ CSV ក្នុងថត (Folder) `results/`។

## ៣. សេចក្តីសន្និដ្ឋាន
ជារួម គម្រោងនេះប្រៀបដូចជាការដាក់ **"ខួរក្បាលឆ្លាតវៃ"** ទៅក្នុងភ្លើងស្តុបនៅភ្នំពេញ។ វាមិនត្រឹមតែដឹងថាមានឡានកកស្ទះទេ ប៉ុន្តែវាដឹងថាស្ទះម៉ូតូច្រើន ឬស្ទះឡានច្រើន រួចប្រើប្រាស់បច្ចេកវិទ្យាកម្រិតខ្ពស់ (Quantum Approach) ក្នុងការសម្រេចចិត្តរំដោះចរាចរណ៍នោះបានយ៉ាងរហ័សនិងមានប្រសិទ្ធភាពជាងវិធីចាស់ៗដែលយើងកំពុងប្រើសព្វថ្ងៃ។
